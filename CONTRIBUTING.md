# Contributing

## Commit Convention

This repository follows [Conventional Commits](https://www.conventionalcommits.org/).

Every commit message must follow the format:

    type(scope): description

### Types

- feat -- new feature
- fix -- bug fix
- docs -- documentation only
- test -- adding or updating tests
- refactor -- code change that is not a fix or feature
- chore -- build, tooling, or dependency changes

### Examples

    feat(node): add swagger-ui-express and OpenAPI spec
    fix(node): update Dockerfile base image for Prisma compatibility
    docs(node): add CONTRIBUTING.md with Conventional Commits convention
    test(node): add integration test suite against containerized instance

This convention applies to all commits, including those generated by automated/agent workflows (Stepwise, once installed).
